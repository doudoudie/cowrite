# 通过 AODW 写代码的实践

## 流程总览

描述需求 → 反思审查 → 决策 → 审计 → 执行 → 开发审计

## 步骤一：描述需求

1. 说出需求的输入、输出、关键逻辑，形成粗略的需求文档

## 步骤二：反思审查

1. 用 AODW 框架以"以终为始"的决策思路审查需求文档 → 输出待决策清单
2. 澄清反馈的待决策清单
3. 将澄清确认的问题补充到需求文档

## 步骤三：决策

1. 生成 RT 结构目录（决策文档、技术方案、影响分析、约束条件）

## 步骤四：审计

1. 用 AODW 框架以"以终为始"的决策思路审计需求文档 → 输出待决策清单和风险
2. 澄清审计的问题清单
3. 将澄清确认的问题补充到需求文档
4. 重复步骤 1-3，持续 3-5 轮完善
5. 得到一份相对完整的需求文档

## 步骤五：执行

1. 用 AODW 框架以"以终为始"思路写完整测试用例（功能独立、单独可测、颗粒度小）
2. 用测试用例反过来验证需求文档的完整性
3. 生成开发计划（分多段，每段分点，逐步执行）
4. 每步完成后用测试用例测试，并确认下一步任务
5. 用边界数据、并发数据、安全数据做测试

## 核心经验

- 需求要说清楚：输入、输出、每一步的逻辑，确认好再动手
- 小步执行：一步成功后及时纠正偏差，不要等到最后才发现需求理解错了
- 测试用例很重要：并发问题、边界问题、安全问题都能从测试用例中发现
- 以终为始：始终盯着最终目标思考每一步的对错
