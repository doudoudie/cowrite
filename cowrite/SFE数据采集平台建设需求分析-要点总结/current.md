# SFE数据采集平台建设需求分析 - 主要观点总结

## 核心问题

1. **平台功能缺失** — 缺乏统一的线上化数据采集平台，数据来源分散、标准不一，无法有效整合分析；无法覆盖从结构化目标管理到非结构化情报收集的全场景需求。

2. **管理闭环断裂** — "目标规划-结果填报"核心链条缺乏系统化支撑，规划与执行脱节，难以过程跟踪与效果评估。

## 预期目标

| 方向 | 要点 |
|---|---|
| **统一采集能力** | 线上化工具 + 支持一次性/周期性任务 + 自定义表单 |
| **模板管理** | 租户级模板管理，保障数据规范与采集效率 |
| **业务闭环** | 目标规划↔结果填报手工关联 + 审批工作流 |

## 典型业务场景

- 📊 **季度目标规划** — 一次性项目，面向全国收集下季度销售目标
- 📅 **月度结果填报** — 每月1日自动触发周期性项目，回收实际销售数据
- 🔗 **规划与填报关联** — 目标审批通过后自动创建关联的月度填报项目
- 📰 **竞品调研/市场情报** — 非结构化数据采集
- 🏥 **重点医院开发进度周报** — 周期性项目

## 用户角色（6类）

| 角色 | 职责 |
|---|---|
| 系统管理员 | 租户级标准模板管理 |
| 业务模板创建者 | 通用表单模板创建维护 |
| 项目创建者 | 项目创建、配置、发布、监控（核心运营者） |
| 数据填报人 | 接收任务、完成数据填报提交 |
| 数据审批者 | 审阅、批复规划数据（地区经理） |
| 数据发布者 | 锁定、发布、下架等高级操作（区域管理部） |

## 业务流程关键节点

**项目创建** → 基础配置 → 选择模板 → 设定范围与角色 → 配置权限 → 细化参数 → **任务下发**

**数据填报** → 创建/编辑/提交/撤回 → 地区经理审批/驳回 → 区域管理部锁定/否决/发布 → **归档闭环**

## 成功标准

- 🕐 **10分钟内** 完成一个数据采集项目创建
- ✅ 支撑目标规划、结果填报、情报收集三大核心场景
- ✅ 实现"规划-执行-审阅"全链路闭环
